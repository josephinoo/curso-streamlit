El Scripting Flow — el corazón de Streamlit 🧠

¿Cómo ejecuta Streamlit tu código?

De arriba → abajo, siempre 🔄

“Este es el concepto más importante de todo el módulo. Entender el scripting flow te va a evitar el 80% de los bugs y confusiones que tienen los principiantes. Presta mucha atención aquí.”

El flujo paso a paso

1
Usuario abre la app
Streamlit ejecuta el script completo de arriba a abajo
↓
2
Usuario interactúa (click, slider, input...)
Streamlit vuelve a ejecutar TODO el script desde cero
↓
3
Streamlit compara el resultado
Solo actualiza en pantalla lo que cambió (DOM diffing)
↓
4
Pantalla actualizada ✅
El usuario ve el nuevo resultado al instante

“Cada vez que alguien hace clic en un botón, mueve un slider o escribe en un campo de texto, Streamlit re-ejecuta tu script completo de arriba a abajo. Esto suena ineficiente pero en la práctica es increíblemente simple y predecible. No hay estado oculto, no hay callbacks complejos.”

La regla de oro 🥇

🔄
Cada interacción del usuario = re-ejecución completa del script
✅ El orden del código importa — lo que está arriba se ejecuta primero
✅ Las variables no persisten entre ejecuciones (a menos que uses st.session_state)
✅ El código Python normal funciona exactamente igual — Streamlit solo agrega widgets

“Graben esto en la memoria: cada vez que el usuario hace algo, el script corre de nuevo de arriba a abajo. Las variables se reinician. Los cálculos se repiten. Esto se puede optimizar con @st.cache_data (módulo 3), pero por ahora lo importante es entender el modelo.”

Widgets de Texto y Títulos 📝

st.title() · st.header() · st.subheader()

📄 código
import streamlit as st

st.title("Título Principal")
# Argumento: texto (str)
# Nivel H1 — el más grande

st.header("Sección")
# Argumento: texto (str)
# Nivel H2 — para secciones

st.subheader("Subsección")
# Argumento: texto (str)
# Nivel H3 — para subsecciones
🌐 resultado
Título Principal
Sección
Subsección

Los tres aceptan anchor (para links internos) y help (tooltip). Ej: st.title("App", anchor="inicio")

“title, header y subheader son equivalentes a H1, H2 y H3 en HTML. El argumento principal es siempre el texto. Opcionalmente puedes pasar anchor para crear IDs en el HTML y help para mostrar un tooltip de interrogante.”

st.write() — el comando todo-en-uno

st.write() acepta casi cualquier cosa:

Negrita y cursiva

42
st.write({"a": 1, "b": 2})

st.write(df)
📊 tabla interactiva

“st.write es el navaja suiza de Streamlit. Detecta automáticamente qué tipo de dato le estás pasando y lo renderiza de la forma más apropiada. Texto → markdown. Número → número. DataFrame → tabla. Gráfico de matplotlib → imagen. Es perfecto para prototipado rápido.”

st.text() · st.markdown() · st.caption()

📄 código
st.text("Texto plano sin formato.")
# Argumento: body (str)
# Sin markdown — literal

st.markdown("**Negrita**, *cursiva*, `código`")
# Argumento: body (str)
# unsafe_allow_html=True → HTML directo

st.caption("Nota al pie o aclaración")
# Argumento: body (str)
# Texto pequeño y gris — ideal para
# fuentes, notas y aclaraciones
🌐 resultado
Texto plano sin formato.
Negrita, cursiva, código
Nota al pie o aclaración

“st.text es para texto sin procesar, literalmente lo que escribes. st.markdown es el más poderoso — acepta Markdown completo e incluso HTML si activas unsafe_allow_html. st.caption es ideal para footnotes, créditos o aclaraciones secundarias — sale en gris y pequeño.”

st.code() · st.divider()

📄 código
st.code("""
def saludar(nombre):
    return f"Hola, {nombre}!"
""", language="python")
# language: "python","sql","bash","json"...
# line_numbers=True → muestra nros de línea

st.divider()
# Sin argumentos
# Solo dibuja una línea separadora
# Equivalente a st.markdown("---")
🌐 resultado

“st.code muestra bloques de código con syntax highlighting. El argumento language acepta cualquier lenguaje que soporte Prism.js. Con line_numbers=True agrega números de línea — muy útil para tutoriales. st.divider es simplemente una línea horizontal para separar secciones visualmente.”

Inputs Simples — El Usuario Habla 🎛️

st.button()

clicked = st.button(
    label="Haz clic aquí",  # texto del botón (str) *obligatorio*
    key="mi_boton",         # ID único (str) — necesario si hay varios
    help="Tooltip al hover",# texto de ayuda (str)
    type="primary",         # "primary" (azul) o "secondary" (gris)
    disabled=False,         # deshabilitar (bool)
    use_container_width=True# ancho completo (bool)
)

if clicked:
    st.write("¡Botón presionado! 🎉")
⚠️ Comportamiento clave:
Devuelve True solo en la ejecución inmediatamente posterior al click
En la siguiente re-ejecución vuelve a False
Para estado persistente → st.session_state

“st.button devuelve True solo una vez — en la re-ejecución del script inmediatamente después del click. Si el usuario no vuelve a hacer clic, en la próxima ejecución devuelve False. Esto confunde a muchos principiantes. Es por el scripting flow: cada ejecución es un nuevo mundo.”

st.checkbox() · st.text_input()

st.checkbox()
activo = st.checkbox(
    label="Activar opción",
    value=False,    # estado inicial (bool)
    key="check1",   # ID único
    help="Ayuda",   # tooltip
    disabled=False  # deshabilitar
)
# Devuelve True/False
st.text_input()
texto = st.text_input(
    label="Escribe algo",
    value="",         # valor inicial (str)
    max_chars=100,    # máx caracteres (int)
    key="input1",     # ID único
    type="default",   # "default" o "password"
    placeholder="..."  # texto fantasma
)
# Devuelve el string escrito

“checkbox devuelve True o False. Es perfecto para mostrar/ocultar secciones condicionales con if. text_input devuelve el string que escribió el usuario. El type='password' oculta los caracteres — muy útil para tokens y contraseñas.”

Todos comparten los mismos argumentos:
body (str) — el mensaje  ·  icon (str) — emoji o None  ·  expanded (bool) — colapsable si es largo

“Los cuatro mensajes de estado tienen la misma firma de función. El único argumento obligatorio es el texto. El color y el ícono izquierdo se determinan automáticamente según el tipo. Puedes pasar icon para personalizar el emoji que aparece a la izquierda.”

⏳ st.spinner()
with st.spinner("Procesando..."):

# text (str): mensaje del spinner

st.columns()

El Scripting Flow en acción — Todo junto 🎯

App completa de ejemplo

📄 app_completa.py
import streamlit as st

st.title("🎛️ Mi App de Demostración")
st.divider()

# Sección de inputs
col1, col2 = st.columns(2)

with col1:
    st.header("Configuración")
    nombre = st.text_input("Tu nombre", placeholder="Ej: Ana")
    edad = st.slider("Edad", 1, 100, 25)
    activo = st.checkbox("Modo avanzado")

with col2:
    st.header("Controles")
    nivel = st.number_input("Nivel", 1, 10, 1)
    if st.button("Generar", type="primary"):
        st.balloons()

st.divider()

# Sección de output — reacciona a los inputs
if nombre:
    st.success(f"¡Hola {nombre}! Tienes {edad} años.")
    if activo:
        with st.expander("Detalles avanzados"):
            st.info(f"Nivel configurado: {nivel}")
            st.code(f"usuario = dict(nombre='{nombre}', edad={edad})")
else:
    st.warning("Ingresa tu nombre para comenzar.")
🌐 resultado
🎛️ Mi App de Demostración
Tu nombre
Edad
Valor: 25
Nivel
Nivel: 1

“Este es un ejemplo realista que combina todo lo que vimos. Noten cómo el output al final del script reacciona en tiempo real a los inputs que están arriba. Eso es el scripting flow en acción — Python de arriba a abajo, reactive sin frameworks complejos.”

¿Por qué el orden importa?

❌ Orden incorrecto
st.write(f"Hola {nombre}")  # ← ERROR
# nombre no existe todavía!

nombre = st.text_input("Nombre")
NameError: name 'nombre' is not defined
✅ Orden correcto
nombre = st.text_input("Nombre")
# ↑ primero definimos el widget

if nombre:
    st.write(f"Hola {nombre}")  # ✅
Funciona perfectamente 🎉

El scripting flow garantiza que el código se lee como un libro: de arriba a abajo. Primero defines, luego usas.

“Esta es la trampa más común en Streamlit. Como el script corre de arriba a abajo, no puedes usar una variable antes de declararla. A diferencia de frameworks con callbacks o estado reactivo, en Streamlit el flujo es completamente lineal y predecible.”
