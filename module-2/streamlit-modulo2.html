Regla de oro: controles, filtros y configuración → sidebar · tablas, gráficos y resultados → centro

El sidebar separa los controles del contenido. Como cualquier app profesional — la barra lateral para ajustes, el centro para ver resultados. Piensen en VS Code, Figma, cualquier app que usen a diario.

Las dos formas de usar el Sidebar

📝 Forma 1 — Prefijo directo
st.sidebar.title("Configuración")
st.sidebar.slider("Año", 2018, 2024)
st.sidebar.selectbox("País", ["USA","China"])
st.sidebar.button("Aplicar")
Simple para 2–3 widgets
ó
📦 Forma 2 — Bloque with ✅
with st.sidebar:
    st.title("Configuración")
    st.slider("Año", 2018, 2024)
    st.selectbox("País", ["USA","China"])
    st.button("Aplicar")
Más limpio cuando crece el sidebar

Las dos formas hacen exactamente lo mismo. La mayoría prefiere with st.sidebar: cuando hay más de 3 widgets porque es más fácil de leer.

No hay diferencia técnica. Yo recomiendo la Forma 2 con with st.sidebar: porque agrupa visualmente todo lo que va en el panel lateral y es más fácil de leer cuando el proyecto crece.

selectbox y radio — El Usuario Elige 🎯

st.selectbox() — Lista desplegable

📄 código
opcion = st.selectbox(
    label="¿Qué quieres ver?",
    options=["Datos", "Gráfico", "Resumen"],
    index=0,     # cuál empieza seleccionado
    key="menu",  # ID único (importante si hay varios)
    help="Elige la vista"  # tooltip con ?
)

# La variable guarda el texto elegido:
st.write(f"Elegiste: {opcion}")
🌐 demo
¿Qué quieres ver?
→ Elegiste: Datos
💡 index=0 → primer elemento por defecto
💡 Devuelve el texto de la opción elegida
💡 key es el ID único del widget

selectbox es un dropdown estándar. Le pasas una lista de opciones, devuelve el texto del que eligió el usuario. index=0 hace que el primero empiece seleccionado. El key es importante cuando tienes varios selectbox en la misma página para que Streamlit los distinga.

st.radio() — Botones de opción visibles

📄 código
pagina = st.radio(
    label="Navegar a:",
    options=["📊 Ver Datos", "📈 Ver Gráfico"],
    index=0,           # opción inicial
    horizontal=False,  # True = en fila horizontal
    key="navegacion"
)

if pagina == "📊 Ver Datos":
    st.write("Mostrando la tabla...")
elif pagina == "📈 Ver Gráfico":
    st.write("Mostrando el gráfico...")
🌐 demo
Navegar a:
📊 Ver Datos
📈 Ver Gráfico
→ Mostrando la tabla...
💡 Ideal para 2–4 opciones
💡 horizontal=True → botones en fila

Radio muestra todas las opciones al mismo tiempo. El usuario ve de un vistazo qué puede elegir sin tener que abrir ningún dropdown. Es el widget ideal para el menú de navegación de nuestra app.

Selectbox vs Radio — ¿Cuándo usar cuál?

selectbox 📋 radio 📻
Opciones visibles Solo la elegida Todas siempre
Cantidad ideal 5 o más ✅ 2 a 4 ✅
Espacio que ocupa Muy poco Más espacio
Úsalo para... Países, años, categorías largas Menú de páginas 👈

Para el menú de navegación → usaremos st.radio en el sidebar. Pocas opciones, siempre visibles.

La regla práctica: si tienes 2 a 4 opciones y quieres que el usuario las vea todas, usa radio. Si tienes una lista larga como 50 países o 10 años, usa selectbox para no ocupar tanto espacio vertical.

st.file_uploader() — el usuario sube el archivo

📄 código
archivo = st.file_uploader(
    label="Sube el CSV de Kaggle",
    type=["csv"],   # solo acepta CSV
    key="uploader",
    help="Descárgalo de Kaggle primero"
)

# ⚠️ SIEMPRE verificar antes de usar
if archivo is not None:
    df = pd.read_csv(archivo)
    st.success("¡Archivo cargado! ✅")
    st.write(df)
else:
    st.info("Sube un archivo para comenzar")

El Menú de Navegación 🗺️

La idea — solo un if/elif

🎛️ El sidebar tiene un st.radio con los nombres de las páginas
↓
📦 La elección queda guardada en la variable pagina
↓
🔀 Un if / elif decide qué contenido mostrar en el centro

No hay magia. Es Python normal. Cuando el usuario cambia el radio → el script corre de nuevo → el if decide qué página mostrar.

Antes de escribir el código, el concepto. Sidebar + radio + if/elif = navegación. Nada más. Es el mismo scripting flow que vimos en módulo 1.

El código del menú — 3 bloques

① El sidebar con el menú

Plotly es la librería de gráficos más popular en Python para datos. Los gráficos son interactivos por defecto — hover para ver valores exactos, zoom con scroll. Se integra perfectamente con Streamlit. En módulo 4 profundizamos.

Tu primer gráfico de barras

import plotly.express as px   # siempre así, arriba

# Datos de ejemplo
datos = {
    "País":    ["China", "Mexico", "Canada", "EU"],
    "Arancel": [25,       15,       7,        20]
}
df_ej = pd.DataFrame(datos)

# Crear el gráfico — 1 línea ✨
fig = px.bar(
    df_ej,
    x="País",        # columna para el eje X
    y="Arancel",     # columna para el eje Y
    title="Aranceles por País",
    color="Arancel",
    color_continuous_scale="Reds"
)

st.plotly_chart(fig, use_container_width=True)

Streamlit detecta automáticamente la carpeta pages/ y crea el menú lateral. Sin código extra de navegación.

Para apps pequeñas, un archivo está bien. Para apps reales con muchas pantallas, la carpeta pages es la opción correcta — cada pantalla en su propio archivo, Streamlit hace el menú solo.

# app.py — portada
import streamlit as st

# ⚠️ DEBE ser el PRIMER comando de Streamlit en el archivo
st.set_page_config(
    page_title="Dashboard Aranceles",
    page_icon="🇺🇸",
    layout="wide"
)

st.title("🇺🇸 Dashboard — Aranceles USA")
st.write("Bienvenido. Usa el menú lateral para navegar.")
st.info("👈 Selecciona una página en el menú de la izquierda")
# pages/1_Datos.py
import streamlit as st
import pandas as pd

st.title("📊 Tabla de Aranceles")
archivo = st.sidebar.file_uploader("Sube el CSV", type=["csv"])

if archivo is None:
    st.info("Sube el archivo CSV desde el panel lateral.")
    st.stop()           # ← para aquí si no hay archivo

df = pd.read_csv(archivo)
col1, col2, col3 = st.columns(3)
col1.metric("Filas",   len(df))
col2.metric("Países",  df["country"].nunique())
col3.metric("Máx %", f"{df['tariff_rate_pct'].max():.1f}%")
st.dataframe(df, use_container_width=True, height=320)
# pages/2_Grafico.py
import streamlit as st
import pandas as pd
import plotly.express as px

st.title("📈 Gráfico de Aranceles")
archivo = st.sidebar.file_uploader("Sube el CSV", type=["csv"])

if archivo is None:
    st.info("Sube el archivo CSV desde el panel lateral.")
    st.stop()           # ← para aquí si no hay archivo

df = pd.read_csv(archivo)
resumen = df.groupby("country")["tariff_rate_pct"].mean().reset_index()

fig = px.bar(resumen.sort_values("tariff_rate_pct"),
    x="tariff_rate_pct", y="country", orientation="h",
    title="Arancel promedio por país",
    color="tariff_rate_pct", color_continuous_scale="Reds",
    labels={"tariff_rate_pct": "Arancel (%)", "country": "País"})
st.plotly_chart(fig, use_container_width=True)

Haz clic en las pestañas. Cada página es independiente — sus propios imports, su propio file_uploader. En módulo 3 vemos session_state para compartir el DataFrame entre páginas sin pedir el CSV dos veces.

st.stop() y st.set_page_config()

❌ Sin st.stop() — app rota
archivo = st.file_uploader("CSV")

# archivo es None al arrancar
df = pd.read_csv(archivo)
# ↑ ERROR: no se puede leer None
st.dataframe(df)
🔴 La app explota al arrancar
✅ Con st.stop() — app robusta
archivo = st.file_uploader("CSV")

if archivo is None:
    st.info("Sube un archivo 👆")
    st.stop()   # ← para aquí

# Solo llega aquí si hay archivo
df = pd.read_csv(archivo)  # seguro ✅
st.dataframe(df)
🟢 Espera pacientemente al usuario

Regla: siempre que dependas de algo que el usuario aún no hizo → st.stop() antes de usarlo. Y set_page_config() siempre va primero en el archivo.

st.stop es como el portero de la disco — si el usuario no trajo el archivo (el carnet), lo para en la puerta elegantemente. Sin él, la app intenta leer None y da un error feo.
